## What landed (no commits run yet — they're below)

### 1. Review File Info preview flow (visualschedulebot UX parity)
The single-photo form now has a **big blue "Review File Info" button** between the file picker and description field, plus a thumbnail preview that appears when you choose a file. Click Review File Info → server reads EXIF + reverse-geocodes via Nominatim + computes Plus Code + builds an auto-description sentence ("This image is from Tuesday, April 21, 2026 at 2:10 PM at RWRJ+2V Watrous, NM, USA") → form prefills with "from EXIF" / "from phone GPS" pills next to each label. You can edit any field before clicking Save Photo. Pure preview, no DB write.

**4 backend files (3 new, 1 modified):**
- `server/code/services/photo_intake/description_template.py` (NEW, 144 LOC) — composes the auto-description sentence with defensive collapsing on missing parts.
- `server/code/services/photo_intake/plus_code.py` (NEW, 162 LOC) — pure-Python Open Location Code encoder. Verified against Google's reference fixture (47.0000625, 8.0000625 → "8FVC2222+22" ✓).
- `server/code/services/photo_intake/geocode_real.py` (NEW, 197 LOC) — Nominatim wrapper using stdlib urllib (no `requests` dep). Fail-soft on network/timeout/parse errors. US state name → 2-letter abbreviation map for visualschedulebot-style address.
- `server/code/api/routers/photos.py` — new `POST /api/photos/preview` endpoint wiring it all together. Returns `{captured_at, gps, plus_code, address, description, raw_exif_keys}`.

**3 frontend files (all modified):**
- `ui/photo-intake.html` — Review File Info button + thumbnail preview area + source-attribution pill spans on description/date/location labels.
- `ui/js/photo-intake.js` — file-picker change → thumbnail preview + button enable; Review File Info handler POSTs to `/api/photos/preview` and autofills fields with attribution pills; **+ 2 inline P1 fixes from the overnight code review** (see #4 below).
- `ui/css/photo-intake.css` — Review button styling + thumbnail preview frame + source-pill colors (blue=EXIF, green=GPS, yellow=curator).

### 2. `docs/PILLOW-VENV-INSTALL.md` (the 90-minute trap doc)
